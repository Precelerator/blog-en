No-Code Tool Glide: Generate mobile-friendly apps from Google Sheets

By Fabio Maienschein
18/09/2020
Spreadsheet software such as Excel and Google Sheets is an essential tool for data collection in a large number of projects, but spreadsheets are often not easy to grasp and tend to be difficult to use on mobile. At the same time, spreadsheets are the most common source of data in all kinds of imaginable projects which is a source of frequent annoyance.
 
This is exactly the use case that the American startup Glide addresses and promises the automatic generation of mobile-capable web apps straight out of Google Sheets. Programming knowledge is not necessary and everything you need is provided by Glide. Examples of Glide apps can be found in the extensive template gallery, which contains demo apps for different use cases (e.g. a conference timetable, a CRM or a task tracker).

Technical Implementation

Glide uses Google Sheets to store the app's data and generates a so-called progressive web app for the user, ultimately nothing more than a mobile web application that smartphone users can install on their home screen. This saves Glide the detour via the respective App Stores, instead Glide can generate a URL which users can use to install the created app. Due to the availability via a link and the execution in the browser, Glide apps run automatically on any mobile device regardless of the operating system (Android, iOS, etc.). After clicking on the respective link, the user is prompted to add the app to the home screen of his smartphone. The trick: since Progressive Web Apps are web applications so the app does not need any storage space (since unlike native apps, it runs directly in the user's browser).
 
A visual editor allows Glide users to customize the look and feel of the app, so that the display and positioning of content can be adjusted as desired. Data can be manipulated in the associated spreadsheet which means that adding, editing and deleting entries can be done within Google Sheets.

Pricing

Glide offers different Pricing-Plans, the Personal Plan with Glide logo and some restrictions (<500 records, <1000 edits per month, <100 MB storage space for pictures etc.) is provided free of charge. Professional apps without branding and with additional features such as upstream authentication, a dedicated domain, up to 25000 records, 10 GB of memory and unlimited edits are available for around 30€ per month.

Resume

Overall, Glide provides an impressive feature set for no-code apps, and with Google Sheets as the primary data storage and Glide's visual app editor, it is especially suited for users who have no programming experience. Due to the technical implementation as a progressive web app, Glide apps are very easy to divide and install, so as a result a presentable application can be created with just a few hours of work. Due to the Glide branding, the free plan is only suitable for more professional projects to a limited extent, but with 30€ per month, the pricing for pro apps is low compared to self-implementation or commissioning a developer with the app development. Glide positions itself prominently in the area of simple list-based apps, but with features such as the display of content such as images or maps, it also offers content that goes beyond pure lists. Whether Glide is a sensible choice definitely depends on the type of application, so it is worth taking a look at the sample applications in order to get a better feel for the features and limitations of Glide.
